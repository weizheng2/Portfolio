Is the project I was assigned to after finishing my internship with the Studio and becoming a full time programmer.

Fight Legends, también conocido como Dark Steel, es un juego de lucha RPG medieval para móvil.
Es el proyecto al que fui asignado después de terminar mis prácticas en el estudio y convertirme en programador a tiempo completo.
Debajo se muestran algunas de mis principales contribuciones:

    General

  • Implement UI screens structure and functionality based on the artist designs, taking into account different resolutions.
  • Ensure any feature or update can be added without disrupting the previous versions. Allowing the coexistence of multiples versions from the same store or different stores.
  • Implemented different A/B testing processes, where the game showed different features or with some variations to test which one performed better.

    Ads and other plugins

  • Added advertisement plugins to show rewarded videos and interstitials. The probability to show is controlled via server.

    Addressables

  • Implemented asynchronous asset loading using Unity Addressables System and a custom Google Play Addressables Plugin, overcoming build size restrictions and enhancing performance.
  • Previously used Amazon Web Services for storing the assets.

    Back System

  • A 'back' system that allows the user to close pop ups, return to the previous screen, pause the game, etc.

    Builds

  • I was in charge of making the builds for all the different stores, ensuring the compliance.
  • Also integrated payment platform Xsolla, to facilitate transactions across different regions and app stores.

    Local/Push Notifications

  • Enabled Local Notifications with Unity Mobile Notifications for scheduled events.
  • Push notifications with Firebase for various instant actions.
  • Also made an internal notifications interface for when the player is currently in game and with the screen focused.

    Multiple Characters

  • Developed a multi-character system, enabling players to create, switch, and manage multiple characters.
  • Since the game was designed for one character per user, I had to make plenty of changes and make sure those changes did not affect the production game/server.

    Referral System

  • Allows players to invite others and earn rewards.

    Events

  • Created the first event in the game. It has a system where players can enter undefeated enemy rooms and continue where other players left.
  • Since multiple players can try to enter the room, I had to make sure to only let the first request in, avoiding any race condition.
  • Additionally, it contains a complete new enemy model and a special shop where players can flip cards to get random rewards.

    IAPs, subscription and bundles

  • For each store, I implemented validations for the receipts received in the server after a player buys any IAP.
  • Also, schedule cron jobs or tasks to update subscription data and process refunded, canceled or not given IAPs.
  • For the bundles, I also had to create conditions for showing and purchasing them, since they appear individually for each player and are not always available.

    Tournaments and rankings

  • Made a ranking system where players obtain points from playing a specific game mode. It rewards special icons for the chat.
  • Players can see the rankings changing in real time, current and last weeks positions and a global ranking.
  • In the server, each time the top ranking changes, it sends a push notification to the affected player.
  • Each week, a cron job is scheduled to process, qualify and reward rankings.

    Tutorial

  • Implemented a comprehensive tutorial system, leading players through core game mechanics, combat actions and main interface screens.
  • In the combat part, I made a step by step guide where the combat only advances if the player performs the required action.
  • For the other parts, the system blocks everything except the buttons or actions we want. The player is shown where to focus with an announcer and with direction arrows.

    Chat system (mods, block, ban etc)

  • Created a chat system where players can either comunicate in global channels or send private messages.
  • Global channels are managed in a server panel, allowing making it easier to create and delete channels.
  • Players can also block/unblock others and if enabled to be a moderator, they have the ability to ban other players.

    Village Mode

  • A village mode, where players can collect and upgrade buildings that provide different boosts.

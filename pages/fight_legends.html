Fight Legends, también conocido como Dark Steel, es un juego de lucha RPG medieval para móvil.
Es el proyecto al que fui asignado después de terminar mis prácticas en el estudio y convertirme en programador a tiempo completo.
Debajo se muestran algunas de mis principales contribuciones:

    General

  • Im in charge of making builds and submitting the game for all the different stores, ensuring compliance with their requirements.
  • Handle and process data, manage complex systems (tournaments, rankings, events) in server.
  • Implement UI screens structure and functionality based on the artist designs, taking into account multiple resolutions.
  • Structure different A/B testing processes, where the game shows different features with some variations to test which one performs better.
  • Ensure any feature or update can be added without disrupting the previous versions. Allowing the coexistence of multiples versions from the same store or different stores.

    General

  • Encargarme de compilar y enviar builds a todas las diferentes tiendas, asegurando el cumplimiento de sus requisitos.
  • Manejar y procesar datos, gestionar sistemas complejos (torneos, rankings, eventos) en el servidor.
  • Implementar la estructura y funcionalidad de las pantallas de interfaz de usuario basándome en los diseños del artista, teniendo en cuenta múltiples resoluciones.
  • Estructurar múltiples procesos de prueba A/B, donde el juego muestra diferentes funcionalidades con algunas variaciones para probar cuál funciona mejor.
  • Asegurar que cualquier funcionalidad o actualización pueda ser añadida sin interrumpir las versiones anteriores. Permitiendo la coexistencia de múltiples versiones de la misma tienda o de diferentes tiendas.

    SDK and Plugins

  • Added advertisement plugins to show rewarded videos and interstitials. The probability to show is controlled via server.
  • Added analytics SDK like GameAnalytics and Firebase for receiving events required by the team.

    SDK y Plugins

  • Añadir plugins de publicidad para mostrar videos recompensados e intersticiales. La probabilidad de mostrarlos está controlada a través del servidor.
  • Añadir SDK de análisis como GameAnalytics y Firebase para recibir eventos requeridos por el equipo.

    Addressables

  • Implemented asynchronous asset loading using Unity Addressables System and a Google Play Addressables Plugin, overcoming Google Plays build size restrictions and enhancing performance by only loading assets when needed.
  • Previously used Amazon Web Services for storing the assets.

    Addressables

  • Implementar la carga asíncrona de assets utilizando el Sistema de Addressables de Unity y un Plugin de Addressables de Google Play, superando las restricciones de tamaño de compilación de Google Play y mejorando el rendimiento al cargar assets solo cuando sea necesario.
  • Anteriormente se utilizaba Amazon Web Services para almacenar los assets.

    Local/Push Notifications

  • Local Notifications with Unity Mobile Notifications for scheduled events, for instace, when the in-game store refreshes.
  • Push notifications with Firebase for instant actions processed in the server. One example of this is when a players receives a new chat message.
  • Also added an internal notifications interface for when the player is currently in game and with the screen focused.

    Notificaciones Locales/Push

  • Notificaciones Locales con Unity Mobile Notifications para eventos programados, por ejemplo, cuando la tienda dentro del juego se actualiza.
  • Notificaciones Push con Firebase para acciones instantáneas procesadas en el servidor. Un ejemplo de esto es cuando un jugador recibe un nuevo mensaje de chat.
  • También añadí una interfaz de notificaciones internas para cuando el jugador está dentro del juego y con la pantalla enfocada.

    Back System

  • The system allows the user to close pop ups, return to the previous screen, pause the game, etc., with a back button in-game or from the device.
  • The structure uses event bus and each screen can implement a different close event if needed.

    Sistema de Back/Retroceso

  • El sistema permite al usuario cerrar ventanas emergentes, regresar a la pantalla anterior, pausar el juego, etc., con un botón de retroceso dentro del juego o desde el dispositivo.
  • La estructura utiliza event bus y cada pantalla puede implementar un evento de cierre diferente si es necesario.

    IAPs, subscription and bundles

  • For each store, I implemented validations for the receipts received in the server after a player buys any IAP.
  • Schedule cron jobs or tasks to update subscriptions data and process refunded, canceled or not given IAPs.
  • For the bundles, I also had to create conditions for showing and purchasing them, since they appear individually for each player and are not always available.

    Compras integradas (IAP), suscripciones y bundles

  • Para cada tienda, implementar validaciones para los recibos obtenidos en el servidor después de que un jugador compre cualquier compra integrada (IAP).
  • Programar cron jobs o tareas para actualizar los datos de las suscripciones y procesar las compras devueltas, canceladas o no entregadas de las IAP.
  • Para los bundles, también tuve que crear condiciones para mostrarlos y comprarlos, ya que aparecen individualmente para cada jugador y no siempre están disponibles.

    Tutorial

  • Implemented a comprehensive tutorial system, leading players through core game mechanics, combat actions and main interface screens.
  • In the combat part, I made a step by step guide where the combat only advances if the player performs the required action.
  • For the other parts, the system blocks everything except the buttons or actions we want. The player is shown where to focus with an announcer and with direction arrows.

    Tutorial

  • Implementar un sistema de tutorial completo, guiando a los jugadores a través de las mecánicas principales del juego, acciones de combate y pantallas de interfaz principales.
  • En la parte de combate, creé una guía paso a paso donde el combate solo avanza si el jugador realiza la acción requerida.
  • Para las otras partes, el sistema bloquea todo excepto los botones o acciones que se desea. Se muestra al jugador dónde enfocarse con un anunciador y flechas de dirección.

    Game Event

  • Created the first event in the game. It has a system where players can enter undefeated enemy rooms and continue where other players left.
  • Since multiple players can try to enter the same room, made sure to only let the first request in, avoiding any race condition.
  • Additionally, it contains a special shop where players can flip cards to get random rewards.

    Evento del Juego

  • Crear el primer evento en el juego. Tiene un sistema donde los jugadores pueden entrar en salas de enemigos no derrotados y continuar desde donde otros jugadores lo dejaron.
  • Dado que varios jugadores pueden intentar entrar a la misma sala, había de asegurarse de permitir solo la primera solicitud, evitando cualquier condición de carrera.
  • Además, contiene una tienda especial donde los jugadores pueden girar cartas para obtener recompensas aleatorias.

    Chat system (mods, block, ban etc)

  • Created a chat system where players can either comunicate in global channels or send private messages.
  • Global channels are managed in a server panel, allowing making it easier to create and delete channels.
  • Players can also block/unblock others and if enabled to be a moderator, they have the ability to ban other players.

    Sistema de chat (moderadores, bloqueo, baneo, etc)

  • Crear un sistema de chat donde los jugadores pueden comunicarse en canales globales o enviar mensajes privados.
  • Los canales globales son gestionados en un panel de servidor, lo que facilita la creación y eliminación de canales.
  • Los jugadores también pueden bloquear/desbloquear a otros y, si están habilitados como moderadores, tienen la capacidad de banear a otros jugadores.

    Game Shop

  • Implement various shop sections where the player can buy different types of resources.
  • Items section, each player has its own catalog of items.
  • Chests section, players can get content with random possibilities.

    Tienda del juego

  • Implementar varias secciones de tienda donde el jugador puede comprar diferentes tipos de recursos.
  • Sección de objetos, cada jugador tiene su propio catálogo de objetos.
  • Sección de cofres, los jugadores pueden obtener contenido con posibilidades aleatorias.

    Tournaments and rankings

  • Made a ranking system where players obtain points from playing a specific game mode. It rewards special icons for the chat.
  • Players can see the current week ranking updating in real time. And also check last weeks positions and the global all time ranking.
  • In the server, each time the top ranking changes, it sends a push notification to the affected player.
  • Each week, a cron job is scheduled to process, qualify and reward rankings.

    Torneos y clasificaciones

  • Desarrollar un sistema de clasificación donde los jugadores obtienen puntos al participar en un modo de juego específico. Premia con iconos especiales para el chat.
  • Los jugadores pueden ver la clasificación de la semana que se actualiza en tiempo real. También pueden consultar las posiciones de la semana pasada y la clasificación global de todos los tiempos.
  • En el servidor, cada vez que cambia la clasificación principal, se envía una notificación push al jugador afectado.
  • Cada semana, se programa una tarea o cron para procesar, calificar y recompensar las clasificaciones.

    Multiple Characters

  • Developed a multi-character system, enabling players to create, switch, and manage multiple characters.
  • Since the game was designed for one character per user, the biggest challenge was to ensure the changes to the server and database did not affect the production game.

    Múltiples Personajes

  • Desarrollar un sistema de múltiples personajes, permitiendo a los jugadores crear, cambiar y gestionar varios personajes.
  • Dado que el juego estaba diseñado para un solo personaje por usuario, el mayor desafío fue garantizar que los cambios en el servidor y la base de datos no afectaran al juego en producción.

    Referral System

  • Allows players to invite others and earn rewards.
  • Each code is generated with 'Hashids', to generate a unique, reversible, and short hash from the user ID.

    Sistema de referidos

  • Permite a los jugadores invitar a otros y ganar recompensas.
  • Cada código se genera con 'Hashids', para generar un hash único, reversible y corto a partir del ID de usuario.

    Village Mode

  • Players can interact and upgrade buildings that provide different boosts.
  • Focused on decoupling the structure to facilitate the management of each building type.

    Modo Villa

  • Los jugadores pueden interactuar y mejorar edificios que proporcionan diferentes bonificaciones.
  • Me centré en la desvinculación la estructura para facilitar la gestión de cada tipo de edificio.
